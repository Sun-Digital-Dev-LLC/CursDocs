## 建立子使用者

首先，前往 **[控制面板](https://panel.curshosting.net)** 並選擇自己的伺服器，然後點選 **Users**，再點選 **New User** 來新增子使用者。

<img src="https://imgur.com/Rt6ZCy8.png" />

然後輸入想要邀請子使用者的 Gmail，然後選擇子使用者需要的權限，權限詳細內容如下方。

<img src="https://imgur.com/RbkVXqT.png" />

## 子使用者給予權限介紹

<Callout title="重要提示" type="info">
  其他沒有說明的權限，是可能不會使用到或權限過大不建議開放，如對以下說明有不明白可前往群組開啟客服單詢問！
</Callout>
### CONTROL (控制台) | 名稱 | 意思 | 用途 | | ------- | ---------- | --------------------------------
| | CONSOLE | 控制台 | 是否允許子使用者在控制台輸入命令 | | START | 開啟伺服器 |
是否允許子使用者開啟伺服器 | | STOP | 關閉伺服器 | 是否允許子使用者關閉伺服器 | |
RESTART | 重啟伺服器 | 是否允許子使用者重啟伺服器 |

### USER (使用者)

| 名稱   | 意思           | 用途                                 |
| ------ | -------------- | ------------------------------------ |
| CREATE | 創建子用戶     | 是否允許子使用者在創建子用戶         |
| READ   | 查閱子用戶權限 | 是否允許子使用者查閱其他子使用者權限 |
| UPDATE | 更改權限       | 是否允許子使用者更改其他子使用者權限 |
| DELETE | 刪除子帳戶     | 是否允許子使用者刪除子使用者         |

### FILE (檔案管理)

| 名稱         | 意思           | 用途                           |
| ------------ | -------------- | ------------------------------ |
| CREATE       | 創建文件       | 是否允許子使用者創建文件       |
| READ         | 打開文件       | 是否允許子使用者打開文件       |
| READ-CONTENT | 打開並閱讀文件 | 是否允許子使用者打開並閱讀文件 |
| UPDATE       | 更改文件內容   | 是否允許子使用者更改文件內容   |
| DELETE       | 刪除文件       | 是否允許子使用者刪除文件       |
| ARCHIVE      | 壓縮文件       | 是否允許子使用者壓縮文件       |
| SFTP         | 上傳文件       | 是否允許子使用者上傳文件       |

### DATABASES (資料庫)

| 名稱          | 意思             | 用途                             |
| ------------- | ---------------- | -------------------------------- |
| CREATE        | 創建資料庫       | 是否允許子使用者創建資料庫       |
| READ          | 查閱資料庫資料   | 是否允許子使用者查閱資料庫資料   |
| UPDATE        | 更改資料庫資料   | 是否允許子使用者更改資料庫資料   |
| DELETE        | 刪除資料庫及資料 | 是否允許子使用者刪除資料庫及資料 |
| VIEW_PASSWORD | 查看密碼         | 是否允許子使用者查看資料庫密碼   |

### ACTIVITY (活動紀錄)

| 名稱 | 意思             | 用途                         |
| ---- | ---------------- | ---------------------------- |
| READ | 查看面板操作記錄 | 是否允許子使用者查看操作記錄 |
